Seed

  • seed-thinking-1.5 (2504.13914)
    • 数据：
      • STEM + Code + Logical puzzle + Non-verifiable problem
    • 训练策略：
      • 预训练价值模型
      • 提出自适应数据分布，平衡不同类型训练数据（不同评分机制）
    • 损失函数：
      • GAE优势计算：1️⃣解耦价值模型和策略模型的lambda超参2️⃣自适应长度
      • （DAPO）clip higher
      • （DAPO）过滤优势为0的样本，进行动态采样
      • （DAPO）token-mean
      • 引入高质量样本的PTX辅助损失

Qwen

  • qwen-3 (2505.09388)
    • 数据：
      • 用于Reasoning RL阶段的查询-验证对必须满足以下四个条件：1️⃣它们未在冷启动阶段使用过2️⃣对冷启动模型而言是可学习的3️⃣尽可能具有挑战性4️⃣覆盖广泛的子领域
    • 训练策略：
      • large batch size + large group size + off-policy
      • 强调使用了保持熵稳定的策略
    • 损失函数：
      • GRPO
    • 除了上述Reasoning RL阶段的策略，我觉得General RL阶段设计的多样化奖励也很有启发性：
      • 基于规则的奖励
      • 基于模型的奖励（含参考答案）
      • 基于模型的奖励（RL*F）

其他模型

  • skywork-or1 (2505.22312) “多阶段自适应熵调度的GRPO收敛方法”
    • 数据：
      • 过滤正确率为0或1的数据
    • 训练策略：
      • 根据上下文长度划分多阶段（显著降低计算成本）
      • 超长问题：不采用优势掩码，直接进行惩罚
      • 高采样温度（temperature = 1），鼓励探索
      • 严格on-policy，每次rollout只进行一次更新
    • 损失函数：
      • 过滤优势为0的样本
      • 更极致的token-mean（从group级到batch级）
      • 根据目标熵自适应Entropy Loss系数（类比KL Loss中的adaptive beta）
      • w/o KL Loss
  • llama-nemotron (2505.00949)
    • 数据：
      • 预处理：通过小模型上pass@8指标过滤数据，筛选高难度数据
      • 课程学习，难度渐进（先固定批量目标通过率，然后填充样本）
    • 训练策略：
      • 标准GRPO
    • 损失函数：
      • 标准GRPO
  • hunyuan-turbos (2505.15431)
    • 数据：
      • 拒绝采样，筛选高质量样本
    • 训练策略：
      • 高采样温度（temperature = 1），鼓励探索
    • 损失函数：
      • （DAPO）过滤优势为0的样本
      • （DAPO）token-mean
  • mimo (2505.07608)
    • 数据：
      • 去除过于简单和过于困难的数据
    • 训练策略：
      • 维护一个easy data pool，以概率0.1从中采样，提高采样效率的同时稳定训练过程（？）
      • temperature = 1，top-p = 1
    • 损失函数：
      • （DAPO）w/o KL Loss
      • （DAPO）clip higher
      • （DAPO）动态采样，过滤优势为0的样本
      • （DAPO）token-mean
    • 奖励：
      • 对于Coding任务，将测试用例划分为不同难度层，不同难度层分别进行奖励
  • magistral (2506.10910)
    • 损失函数：
      • （DAPO）w/o KL Loss
      • （DAPO）clip higher
      • （DAPO）过滤优势为0的样本
      • （DAPO）token-mean
      • 优势计算：先在组内计算centered score，再在batch内归一化（sequence level）（类似VERL实现的reinforce+±baseline，前者batch内归一化是sequence level，后者是token level）
    • 奖励：
      • 格式奖励 + 正确性奖励 + 长度惩罚（DAPO） + 语言一致性奖励
  • kimi-k2 (2507.20534)
    • 数学：
      • 数据工作：1️⃣数据多样性2️⃣难度适中（基于SFT模型的pass@k）
    • 复杂指令跟随：
      • 数据工作：专家标注 + LLM生成 + LLM数据增强
      • 奖励工作：规则奖励 + LLM评审 + hack-check
    • Faithfulness：
      • 奖励工作：“评判模型能够有效识别那些在上下文中缺乏支持证据却提出事实性断言的句子”
    • 代码：
      • 数据工作：数据集 + 单元测试基建
    • Safety (jail-break)：
      • 数据工作：数据合成
    • 不可验证任务：
      • 奖励工作：self-critic
    • 其他：
      • 超长惩罚
      • PTX辅助损失防止遗忘高质量数据
      • 温度衰减
  • glm-4.5 (2508.06471)
    • 数据：
      • 两阶段课程学习（第二阶段难度更大、数据质量更高）
      • Science domain：注重数据集质量
    • 训练策略：
      • 输出长度固定
      • 基于一个验证集，动态调节采样温度
    • 损失函数：
      • GRPO (w/o KL Loss)
      • Coding domain：token-mean
