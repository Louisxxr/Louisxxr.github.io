Vanilla transformer

Architecture

  1. Normalization位置

    • 绝大部分LLM采用Pre-norm（以及最后输出头之前的Normalization），维护通畅的残差信号路径.
    • Pre-norm保留了恒等梯度路径，使得梯度流动更稳定，因此能支撑大规模网络的稳定训练，并允许使用更大的学习率.
    • 所以有人提出Double-norm、Non-residual post-norm.

  2. LayerNorm还是RMSNorm

    • LayerNorm
      $$y = \frac{x - \mathrm{E}[x]}{\sqrt{\mathrm{Var}[x] + \epsilon}} * \gamma + \beta$$
    • RMSNorm
      $$y = \frac{x}{\sqrt{|x|^{2}_{2} + \epsilon}} * \gamma$$
    • 性能接近，但绝大部分LLM采用RMSNorm.
    • 尽管RMSNorm相较LayerNorm带来更小的显存占以及更少的FLOPs，但是这些提升基本可以忽略（Matmul matters），更重要的是，RMSNorm通过减少显存和计算单元之间的数据传输，有效减少Runtime.

  3. 偏置可弃

    • For FLOPs, memory (though negligible) and optimization stability.

  4. LU和GLU

    • ReLU
      $$FF(x) = \max(0, xW_{1})W_{2}$$
    • ReGLU
      $$FF(x) = (\max(0, xW_{1})\otimes xV)W_{2}$$
    • GeLU
      $$FF(x) = \mathrm{GeLU}(xW_{1})W_{2}$$
      $$\mathrm{GeLU} := x\Phi(x)$$
      $$\Phi(x):\mathrm{\ CDF\ of\ N}(0, 1)$$
    • GeGLU
      $$FF(x) = (\mathrm{GeLU}(xW_{1})\otimes xV)W_{2}$$
    • Swish
      $$FF(x) = \mathrm{Swish}(xW_{1})W_{2}$$
      $$\mathrm{Swish} := x\sigma(x)$$
    • SwiGLU
      $$FF(x) = (\mathrm{Swish}(xW_{1})\otimes xV)W_{2}$$
    • 门控线性单元（GLU）被实验证明能够提升模型性能，通过一个线性变换分支提供门控信号，增强模型表达能力.
    • 为了保持总参数量不变，使用GLU的Feedforward层其Feedforward-dim为原来的2/3.

  5. Parallel layers

    • Serial layers
      $$y = x + \mathrm{MLP}(\mathrm{Norm}(x + \mathrm{Attention}(\mathrm{Norm}(x))))$$
    • Parallen layers
      $$y = x + \mathrm{MLP}(\mathrm{Norm}(x)) + \mathrm{Attention}(\mathrm{Norm}(x))$$
    • 并行化越好，GPU效率越高，如果采用Parallel layers架构，Attention层和FF层的Normalization和Matmul可以融合.

  6. RoPE

    • 正余弦位置编码：在Embedding的不同维度对上使用不同频率的正弦和余弦函数，使得序列位置在这些函数所张成的二维平面上形成沿圆周变化的相位表示.
      $$\mathrm{PE}(pos, 2i) = \sin{\frac{pos}{10000^{\frac{2i}{d_{model}}}}}$$
      $$\mathrm{PE}(pos, 2i + 1) = \cos{\frac{pos}{10000^{\frac{2i}{d_{model}}}}}$$
      $$\mathrm{Embed}(x, i) = \mathrm{TokenEmbed}(x) + \mathrm{PE}_{i}$$
    • 绝对位置编码：可学习.
      $$\mathrm{Embed}(x, i) = \mathrm{TokenEmbed}(x) + \mathrm{PosEmbed}(i)$$
    • 相对位置编码：可学习，作用在Attention层.
      $$e_{ij} = \frac{x_{i}W^{Q}W^{K\top}x_{j}^{\top} + r_{i - j}}{\sqrt{d_{k}}}$$
    • RoPE
      • Preliminary $$ R(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix} $$ 这个变换表示在二维平面上将向量(x, y)绕原点(0, 0)按逆时针方向旋转角度θ，长度保持不变.
      • 公式 $$ R(i; \theta_{1}, \theta_{2}, ..., \theta_{d / 2}) = \begin{bmatrix} \cos i\theta_{1} & -\sin i\theta_{1} & 0 & 0 & ... & 0 & 0 \\ \sin i\theta_{1} & \cos i\theta_{1} & 0 & 0 & ... & 0 & 0 \\ 0 & 0 & \cos i\theta_{2} & -\sin i\theta_{2} & ... & 0 & 0 \\ 0 & 0 & \sin i\theta_{2} & \cos i\theta_{2} & ... & 0 & 0 \\ ... & ... & ... & ... & ... & ... & ... \\ 0 & 0 & 0 & 0 & ... & \cos i\theta_{d / 2} & -\sin i\theta_{d / 2} \\ 0 & 0 & 0 & 0 & ... & \sin i\theta_{d / 2} & \cos i\theta_{d / 2} \end{bmatrix} $$ $$e_{ij} = \frac{x_{i}W^{Q}R(i)(x_{j}W^{K}R(j))^{\top}}{\sqrt{d_{k}}} = \frac{x_{i}W^{Q}\overbrace{R(i)R(j)^{\top}}^{R_{i - j}}W^{K\top}x_{j}^{\top}}{\sqrt{d_{k}}}$$
        满足$f(x, i)\odot f(y, j) = g(x, y, i - j)$.
      • 模型训练和推理时，仍然是计算点积（而不将上式展开），推理时KV cache中的K带位置信息.

  7. MQA / GQA

    • 动机：减少推理时的KV cache存储和内存带宽压力，权衡推理效率和模型性能.
    • Preliminary：Attention维度变化分析.
      • 假设：B(batch-size), N(seq-len), D(model-dim), h(num-heads), d(head-dim)
      • Input $x$: (B, N, D)
      • Parameters $W^{Q}, W^{K}, W^{V}, W^{O}$: (D, h * d)
      • $Q, K, V$: (B, N, h * d) -> (B, N, h, d) -> (B, h, N, d)
      • $Q \cdot K^{\top}$: (B, h, N, N)
      • $\mathrm{softmax}\left(\frac{Q \cdot K^{\top}}{\sqrt{d}}\right) \cdot V$: (B, h, N, d) -> (B, N, h * d)
      • output: (B, N, D)
    • 训练时MACs（Multiply-ACcumulate次数）：
      $$3BND^{2} + BN^{2}D + BN^{2}D + BND^{2} = 4BND^{2} + 2BN^{2}D$$
    • 推理时MACs：
      $$4BND^{2} + 2B(1 * \sum\limits_{n=1}^{N}n)D \approx 4BND^{2} + BN^{2}D$$
    • 训练时访存：
      $$O(\overbrace{BND}^{activations} + \overbrace{D^{2}}^{parameters}+ \overbrace{BhN^{2}}^{softmax})$$
    • 推理时访存：
      $$O(\overbrace{BN^{2}D}^{activations} + \overbrace{ND^{2}}^{parameters})$$
    • 引入MQA，推理时访存：
      $$O(\overbrace{BN^{2}d}^{activations} + \overbrace{ND^{2}}^{parameters})$$
    • 计算强度（Arithmetic intensity）等于总计算量除以总访存量，其临界值为峰值算力除以显存带宽（反映硬件能力），高于临界值说明算力为瓶颈（假设计算量打满，带宽还有余），低于临界值说明带宽为瓶颈（假设访存量打满，算力还有余）.
    • MQA一定程度上减少了推理时的内存带宽压力（但不多）.

  8. Sparse attention (Especially, sliding window attention)

    • 动机：Attention平方复杂度的背景下，对长序列权衡计算代价和模型性能.
    • SWA：让模型深度来扩展有效上下文.
    • 更新的做法：在模型各层交叉使用Full attention和Sparse attention.

      Cohere Command A

      • Interleave 3 SWA blocks and 1 FA block.
      • SWA with RoPE, FA without PE.

Training: hyperparameters (Very empirical and isn’t written in stone)

  1. Feedforward-dim to model-dim ratio
    $$d_{ff} = 4d_{model}$$
    $$\mathrm{Especially\ with\ GLU:\ }d_{ff} = \frac{8}{3}d_{model} = 2.66d_{model}$$

  2. Head-dim * num-heads to model-dim ratio
    $$d_{head} * n_{head} = d_{model}$$

  3. Model-dim to num-layers ratio (Aspect ratio)

    • 约为100.
    • 从并行化的角度考虑，更宽的模型比更深的模型对流水线并行更友好.
    • Residual解决了梯度问题，但大模型时代硬件和并行约束变得更关键，因此宽比深更优.
    • 深度依然存在（100–200比例已经算深了），但不会无限追求堆层.

  4. 词典大小

  5. 需不需要正则

    • 预训练一般只进行1个Epoch，基本不存在过拟合问题.
    • 但是一些模型仍然保留Weight decay策略，有研究说明Weight decay和LR衰减相互作用可以带来更好的Training loss下降.

Training: stability

  1. 什么是“训练稳定性”

    Don’t train models that look like the blue curve!

    • 指标1：Loss.
    • 指标2：梯度范数.
    • 训练稳定性的核心是激活值尺度和参数尺度，问题常常出在Softmax：CE损失没有约束Logits尺度导致Softmax中的指数操作出现上溢或下溢，进而影响训练稳定性.

  2. For output softmax: Z-loss
    $$L_{total} = L_{CE} + L_{Z} = -\log\frac{e^{z_{y}}}{\sum_{j}e^{z_{j}}} + \alpha\left(\log\sum_{j}e^{z_{j}}\right)^{2}$$

    • 当$\sum_{j}e^{z_{j}} = 1$时最优.

  3. For attention softmaxes: Normalization always works

    • 经典的$\sqrt{d_{k}}$.
    • 更新的做法：线性变换得到Q和K之后先进行Normalization.
    • 其他做法：通过Tanh对Logits进行软上限处理，使其不超过某个最大值.
      $$\mathrm{logits} ← \mathrm{softcap} * \tanh(\mathrm{logits} / \mathrm{softcap})$$

02_Tensor theory, Resource accounting, Practical advice

Napkin math

  1. 1024 H100集群，15T tokens，训练一个70B模型，需要多少天？

    $\mathrm{FLOPs} = 6 * 70\mathrm{e}9 * 15\mathrm{e}12$

    $\mathrm{We\ know\ that:\ H100\ FLOPS\ can\ be\ }1979\mathrm{e}12 / 2$

    $\mathrm{We\ assume\ that:\ }\mathrm{MFU} = 0.5$

    $\mathrm{FLOPs\ per\ day} = \mathrm{FLOPS} * \mathrm{MFU} * 1024 * 60 * 60 * 24$

    $\mathrm{days} = \mathrm{FLOPs} / \mathrm{FLOPs\ per\ day}$

  2. 8 H100集群，使用Adam优化器，能训练多大模型？（Naively）

    $\mathrm{bytes\ per\ parameter} = 4 + 4 + (4 + 4)$

    $\mathrm{We\ know\ that:\ H100\ memory\ can\ be\ }80\mathrm{e}9$

    $\mathrm{num\ of\ parameters} = (8 * \mathrm{memory}) / \mathrm{bytes\ per\ parameter}$

    • 对每个参数，需要缓存参数值、梯度值、优化器状态，这里不考虑激活值（激活值显存占用依赖batch_size和seq_len）.
    • 以上计算考虑使用FP32进行训练，如果采用混合精度训练，则动态的参数值和梯度值分别占2字节，同时维护4字节的参数值，显存占用不变，速度变快.

Tensor theory

  1. 我们所定义的Tensor包含指向连续空间的指针和规定读取方式的Metadata.

  2. 注意区分Tensor深复制和浅复制.

  3. 积累常用的Tensor创建、GPU相关、Matmul、Elementwise、Reduce、元素重排操作.

    • 广播规则

      假设两个张量 A 和 B，它们的形状分别是：A.shape = (a1, a2, ..., an) 和 B.shape = (b1, b2, ..., bm)

      1. 对齐维度
        • 从最后一维开始对齐（右对齐）
        • 不足的维度用 1 补齐
      2. 逐维比较，对齐后的每一维 d：
        • 如果 A.shape[d] == B.shape[d] → 保持
        • 如果 A.shape[d] == 1 → A 在该维度广播
        • 如果 B.shape[d] == 1 → B 在该维度广播
        • 如果都不等且都不为 1 → ❌ 报错
      3. 输出形状
        • 每一维的长度 = max(A.shape[d], B.shape[d])
    • 张量矩乘规则
      1. 最后两维总是做标准矩阵乘法：m×k @ k×n → m×n
      2. 前面所有维度做广播：
        • 如果 A 和 B 的前面维度不一样，但可以通过广播规则对齐，则广播后再做矩阵乘
        • 输出 shape = broadcasted_prefix + (m, n)

  4. 维度操作语言Einops

Compute accounting

More: Transformer FLOPs

  1. 一次浮点数加法或乘法即一个浮点数操作（FLOP），模型计算量单位为FLOPs，硬件计算性能单位为FLOPS（每秒可以执行浮点数操作的次数）.

  2. 基本Tensor操作的FLOPs：

    • 一次线性变换（假设一个B x D矩阵x和一个D x K矩阵w进行矩阵乘法）FLOPs为2 * B * D * K，可以解释为对每个(i, j, k)都需要一次x[i][j] * w[j][k]的浮点数乘法和一次浮点数加法.
    • 一个m x n矩阵一次Elementwise操作FLOPs为O(mn).
    • 矩阵乘法占计算量的主要部分，因此，模型前向传播的FLOPs近似为矩阵乘法的FLOPs.

  3. 2 * B * D * K的结论可以推广到Transformer和LLM，因此，一次前向传播FLOPs为2 * (#tokens) * (#parameters).

  4. 引入MFU（Model FLOPs utilization）的概念：

    • $\mathrm{MFU} = \mathrm{actual\ FLOPS} / \mathrm{promised\ FLOPS}$.
    • FLOPS取决于硬件和数据类型（精度）.

  5. 理解反向传播和计算图.

    • 在反向传播中，每个张量的梯度的形状总是与该张量本身的形状完全一致.

  6. 在反向传播中，矩阵乘法同样占计算量的主要部分，以这个简单模型为例：

    • 假设x形状为B x D，w1形状为D x D，w2形状为D x K.
    • w2.grad[j, k] = sum_i h1[i, j] * h2.grad[i, k]，计算w2的梯度FLOPs为2 * B * D * K.
    • h1.grad[i, j] = sum_k w2[j, k] * h2.grad[i, k]，计算h1的梯度FLOPs为2 * B * D * K.
    • 因此，对第2个矩乘做反向传播需要的FLOPs为4 * B * D * K，对第1个矩乘做反向传播同理为4 * B * D * D（尽管不需要输入的梯度）.
    • 推广到LLM，一次反向传播FLOPs为4 * (#tokens) * (#parameters)，因此，训练所需FLOPs为6 * (#tokens) * (#parameters)（预训练一般只进行1轮）.

Practical advice

  1. Parameter initialization

    • Xavier初始化：为了保证训练稳定性（避免梯度爆炸和梯度消失），我们希望模型各层激活值的方差和模型输入保持一致，经过推导，可以得到模型线性变换参数标准差应为输入维度的倒数开方.
    • Attention操作中除以$\sqrt{d_{k}}$一定程度上和这里的原理相同.

  2. Note about randomness

    • 参数初始化、数据集处理、Dropout等过程都会引入随机性，在调试代码时，建议设置随机种子.

  3. Data loading

    • memmap + pin_memory + non_blocking=True
    • This allows us to do two things in parallel:
      • Fetch the next batch of data into CPU.
      • Process x on the GPU.

  4. Checkpointing

    • 训练过程中，定期保存（且只需保存）模型参数和优化器状态.

  5. Mixed precision training

    • 前向传播过程使用BF16或FP8（For activations），其他过程使用FP32（For parameters, gradients, optimizer states）.

Tokenizer

  1. 分词本质是将一段文本转换为整数序列，一个整数即一个Token，整数范围即词典大小.

  2. Character-based tokenization

    • 对于Unicode字符，对应码点即为分词结果.
    • 问题1：在字符级别进行分词，不可避免地都会导致非常大的词典规模（你必须至少为每个字符单独编码）.
    • 问题2：很多字符非常罕见，而这是一种对词典的低效利用.

  3. Byte-based tokenization

    • 通过Unicode编码将字符序列表示为字节序列.
    • 问题：压缩比（每个Token平均表示的字节数）为1，带来长序列问题.

  4. Word-based tokenization

    • 在训练数据上通过正则、统计方法或概率模型Fit出一个分词器.
    • 问题：这个分词器大概率不完备，需要引入<UNK> token.

  5. Byte Pair Encoding (BPE)

    • 通过Unicode编码将字符序列表示为字节序列，然后进行字节对合并.
